# 🎯 最简单解决方案

## 问题
您没有Mac电脑，但需要打包iOS应用。

## ✅ 最佳解决方案

### 方案：使用AWS Mac Instances（按需付费）

**为什么推荐**：
- ✅ 只需要使用2-3小时
- ✅ 成本低（约$2-5）
- ✅ 不需要长期订阅
- ✅ 性能好，速度快

## 🚀 完整步骤（约1-2小时）

### 第一步：注册AWS账号（5分钟）
1. 访问：https://aws.amazon.com/cn/free/ （中文版）
   或：https://aws.amazon.com/ （英文版）
2. 点击"创建免费账户"
3. 填写信息并完成身份验证
4. ⚠️ 需要信用卡（免费套餐不会收费，除非超出限制）

### 第二步：启动Mac实例（15分钟）
1. 登录AWS控制台：https://console.aws.amazon.com/
2. 在顶部搜索框输入 "EC2"
3. 点击进入EC2服务
4. 点击 "启动实例"
5. 在搜索框输入 "mac" 或 "macOS"
6. 选择 "macOS" 实例类型
7. 选择最小的配置即可（mac1.metal）
8. 配置其他设置（使用默认即可）
9. 点击 "启动实例"
10. 等待实例启动（约10-15分钟）

### 第三步：连接到Mac（5分钟）
1. 在EC2控制台找到您的Mac实例
2. 获取连接信息
3. 使用SSH或远程桌面连接

### 第四步：上传项目（10分钟）
**方法1：使用Git**（如果项目在Git仓库）
```bash
git clone <your-repo-url>
cd re_order
```

**方法2：使用文件传输**
- 将项目文件夹压缩
- 通过SCP或远程桌面上传

### 第五步：运行自动打包脚本（30分钟）
```bash
cd client
chmod +x auto-build-ios.sh
./auto-build-ios.sh
```

脚本会自动完成所有准备工作。

### 第六步：在Xcode中打包（20分钟）
按照脚本提示：
1. 配置签名（选择开发团队）
2. Product → Archive
3. Distribute App → Ad Hoc
4. 导出.ipa文件

### 第七步：下载.ipa文件（5分钟）
- 通过远程桌面下载
- 或上传到云存储

### 第八步：关闭实例（节省费用）
在AWS控制台停止实例。

## 💰 成本
- **AWS Mac实例**：约$1-2/小时
- ⚠️ **重要**：Mac实例有24小时最小计费周期
- **实际费用**：约$24-48（即使只使用1小时，也会按24小时收费）
